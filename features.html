Planned Features

The LogiJS project is in a state where many important features of a logic simulator are integrated but we're planning to implement a lot of new stuff to make LogiJS more powerful and easier to use. These are some features that we want to integrate in the future, in no specific order:

  • Copying parts of a sketch: At the moment, LogiJS gives you the ability to select parts of a sketch. These parts can be dragged to another place or be deleted from the sketch. We want to extend this features to enable the user to copy parts of a sketch and paste it multiple times.

  • Server environment to save sketches online: To load your own sketches in LogiJS, you need to get a local copy from GitHub to open the file. This isn't very user friendly and may lead to confusion about how to load sketches. We want to set up a server structure that allows us to handle user accounts and files online. As an alternative to hosting all files ourselves, we could imagine a cloud service based solution like loading files from Google Drive or Dropbox directly into LogiJS.

  • Support for the VHDL hardware description language: VHDL is a language to describe logical circuits, like the one you can create in LogiJS, in a standardized form, for example to execute them on an FPGA. Having an option to im- and export VHDL code may be an important feature for the use of LogiJS in education.

  • Reorganize the components in the toolbox: The toolbox on the left side may be confusing for beginners and some similar components could be accessed with just one button, for example components that exist with various bit widths. Also, some basic components are missing and should be added as standard components. It would be nice to have a selection of your own components under the standard ones. This requires having individual user accounts.

Got ideas for upcoming features?

Just open an issue on GitHub or get in contact with us directly!
